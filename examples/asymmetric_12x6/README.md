# Asymmetric 12x6 Example

Демонстрационный сценарий с той же расчетной сеткой, что и стандартный конфиг: `12 x 6 x 4`.

Главная идея примера - специально сделать теплицу несимметричной, чтобы в терминальной карте было легче увидеть разницу зон.

## Что отличается

- Сильный обогреватель `1500 W` стоит ближе к левому нижнему сектору.
- Второй обогреватель слабее, `700 W`, и смещен к правой верхней части.
- Левая форточка открыта сильно, `0.55`, поэтому рядом с ней зона быстрее приближается к внешней погоде.
- Правая верхняя форточка открыта слабее, `0.15`.
- Увлажнитель смещен к верхней половине теплицы, а не стоит строго по центру.
- Растения стоят несимметрично, включая дальний холодный угол.

## Запуск

Linux/PuTTY:

```bash
./build/greenhouse3d simulate examples/asymmetric_12x6/config.json
```

Windows PowerShell:

```powershell
.\build-mingw\greenhouse3d.exe simulate examples\asymmetric_12x6\config.json
```

Результаты сохраняются в:

```text
outputs/examples/asymmetric_12x6/
```

## Как читать карту

В этом примере отображается:

```json
"field": "temperature"
```

Значит символы и цвета показывают температуру.

```text
low  . : - = + * # % @  high
```

- `.` и `:` - холоднее.
- `-`, `=`, `+` - средняя зона.
- `*`, `#`, `%`, `@` - горячее.

Цвет повторяет тот же смысл: голубой/синий - холоднее, белый - середина, желтый/красный - горячее.

Устройства печатаются поверх температуры:

```text
P plant | H heater | V vent | M humidifier | * multiple devices
```

Если в клетке стоит `H`, виден `H`, а не символ температуры. Температуру около устройства смотрите по соседним символам.

Если символ выглядит жирнее, это не отдельный показатель. Это эффект PuTTY от яркого ANSI-цвета, чаще всего для `*` или ярких устройств.

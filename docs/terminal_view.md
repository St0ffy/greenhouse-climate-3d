# Terminal View

Терминальная визуализация работает в режиме `simulate` и показывает анимацию одного слоя 3D-сетки теплицы.

Запуск:

```bash
./build/greenhouse3d simulate configs/default_config.json
```

На Windows PowerShell с MinGW-сборкой:

```powershell
.\build-mingw\greenhouse3d.exe simulate configs\default_config.json
```

## Настройки

Блок находится внутри `output`:

```json
"terminal_view": {
  "enabled": true,
  "field": "temperature",
  "layer_z": 0,
  "display_stride_x": 1,
  "display_stride_y": 1,
  "frame_interval_seconds": 600,
  "sleep_ms": 150,
  "use_colors": true,
  "clear_screen": true,
  "loop_playback": false,
  "show_devices": true,
  "project_devices_to_layer": true,
  "auto_scale": false,
  "fixed_min_value": 0.0,
  "fixed_max_value": 40.0
}
```

## Поле карты

`field` выбирает отображаемую величину:

- `temperature` - температура ячеек.
- `humidity` - влажность ячеек.
- `temperature_error` - отклонение температуры ячейки от средней целевой температуры растений.

Для температуры удобно использовать диапазон `0..40`, для влажности `0..100`, для ошибки температуры `0..10`.

## Слои

`layer_z` выбирает слой по высоте:

- `0` - нижний слой, ближе к растениям и обогревателям.
- `1` или `2` - средние слои.
- `3` - верхний слой для сетки `nz = 4`, ближе к крыше и форточкам.

Если устройство находится на другом слое, параметр `project_devices_to_layer: true` все равно покажет его на карте по координатам `x/y`. Это удобно для вида сверху.

## Размер карты

Для больших сеток можно уменьшить карту в терминале:

```json
"display_stride_x": 5,
"display_stride_y": 5
```

Тогда один символ в терминале показывает блок `5 x 5` ячеек. Температура или влажность усредняется по блоку, а устройства `P/H/V/M` все равно показываются, если попали внутрь блока.

## Символы

Значения ячеек переводятся в шкалу:

```text
low  . : - = + * # % @  high
```

Для `field: "temperature"` это температура: `.` и `:` холоднее, `-`, `=`, `+` ближе к середине диапазона, `*`, `#`, `%`, `@` горячее.

Для `field: "humidity"` та же шкала показывает влажность: слабые символы означают более сухие зоны, сильные - более влажные.

Для `field: "temperature_error"` шкала показывает ошибку от целевой температуры растений: слабые символы лучше, сильные означают, что зона дальше от цели.

Устройства печатаются поверх карты:

```text
P plant
H heater
V vent
M humidifier
* multiple devices
```

Если в клетке есть устройство, его буква заменяет символ температуры или влажности. Значение около устройства нужно читать по соседним символам.

Цвет повторяет смысл выбранного поля:

- голубой/синий - низкое значение;
- белый - среднее значение;
- желтый/красный - высокое значение.

Цвета устройств отдельные: растение зеленое, обогреватель красный, форточка голубая, увлажнитель фиолетовый, несколько устройств `*` - ярко-белые.

Отдельной метрики "жирность шрифта" нет. Если символ кажется жирнее, это эффект PuTTY от яркого ANSI-цвета, а не дополнительное физическое значение.

## PuTTY

Если цвета отображаются неправильно:

```json
"use_colors": false
```

Если нужно увидеть все кадры подряд, а не анимацию на одном месте:

```json
"clear_screen": false
```

Если нужно зациклить анимацию до нажатия `Esc`:

```json
"loop_playback": true
```

В интерактивном терминале PuTTY клавиша `Esc` останавливает анимацию без нажатия Enter. После этого программа продолжает обычный сценарий: экспортирует файлы и печатает отчет.

Скорость анимации:

- `frame_interval_seconds` - какой шаг симуляционного времени показывать.
- `sleep_ms` - реальная задержка между кадрами в терминале.

## Большая сетка

Для проверки крупной карты есть отдельный пример:

```bash
./build/greenhouse3d simulate examples/large_terminal_view/config.json
```

В нем расчетная сетка `100 x 50 x 4`, длительность `43200` секунд, а `loop_playback` включен. Для терминала карта прореживается через `display_stride_x: 5` и `display_stride_y: 5`, поэтому на экране видно примерно `20 x 10` символов вместо `100 x 50`. CSV в этом примере отключен, чтобы не создавать огромный файл истории всех ячеек.

## Архитектура

`TerminalView.cpp` только читает готовый `SimulationResult`. Он не меняет физику, сетку, устройства и экспорт.

Правильная цепочка в `simulate`:

```text
Simulator -> TerminalView -> Exporter -> Report
```
